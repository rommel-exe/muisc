Key Architectural Decisions

🔄 Instant-Return Proxy

resolve() returns a proxy URL immediately with placeholder metadata. Audio connects to localhost (~10ms) while yt-dlp resolves the CDN URL in parallel — the audio pipeline and URL resolution run concurrently, not sequentially.

🎹 Dual Audio Elements

Two hidden <audio> elements: ACTIVE (playing) and STANDBY (preloading next). On transition, they swap roles instantly — zero-latency gapless playback.

🐍 Persistent Python Daemon

A dedicated Python process keeps yt-dlp's module imports warm (~945ms saved per call). Reads video IDs from stdin, writes stream URLs to stdout. Auto-restarts on crash.

🎯 Duration-Gated Matching

Track identity matching uses duration as the gatekeeper: ±2s = 60% of max score. Forces the EXACT YouTube upload matching the Spotify track's duration, rejecting remixes, covers, and live versions.

🔍 Multi-Query Search Strategy

Up to 9 search queries tried in order (standard, official, explicit, topic, remix, etc.) until confidence threshold ≥ 0.65 is met. Scores improve with each query, early exit when found.

💾 Prewarm Buffer

First 1MB of each audio stream is cached in RAM. The proxy serves this instantly to the browser, then fetches the remainder from CDN via a Range request — sub-second cold start.

🔀 Fisher-Yates Shuffle: When shuffle activates, a pre-computed Fisher-Yates shuffled index array is built from the remaining tracks (after current index). parallelMap(tracks, matchFn, CONCURRENCY):
  results = new Array(tracks.length)
  nextIndex = 0

  worker():
    while nextIndex < tracks.length:
      idx = nextIndex++
      results[idx] = await matchFn(tracks[idx], idx)

  // Spawn CONCURRENCY workers, await all

Unicode Normalization

The normalizeForMatch() function enables fuzzy matching across Unicode representations:

normalizeForMatch("Beyoncé") → "beyonce"
normalizeForMatch("Mötley Crüe") → "motley crue"

Implementation:
1. toLowerCase()
2. normalize('NFD') — splits "é" into "e" + combining accent
3. replace /[\u0300-\u036f]/g — strips combining marks
4. replace /[^a-z0-9\s\-'._]/g — keep only safe characters
5. trim()

18.8 — Engineering Practices

Observations and recommendations for the development process itself, based on patterns visible in the codebase.

PracticeCurrent StateRecommendation
Commits Infrequent, large batches Aim for one commit per logical change. Each commit should compile and pass type checks. This enables clean reverts and bisection.
Testing 4 test files, mix of unit + integration Adopt TDD for all new engines/services. Backfill tests for existing modules before refactoring them. Target 60%+ coverage on application layer, 30%+ overall.
TypeScript Strictness strict: true in tsconfig Excellent foundation. Add noUncheckedIndexedAccess and exactOptionalPropertyTypes to catch class of bugs. Never use as any — current codebase is clean.
Error Handling Mixed — some services have typed errors, others throw generic Standardize on typed error classes per module (already done in proxy/yt-dlp). Extend the pattern to all services. No empty catch {} blocks.
State Management React useState + useReducer in App.tsx For Phase 4, consider React Context + useReducer per domain (QueueContext, PlaylistContext) or a lightweight store. Avoid adding a state library — the app is simple enough that context is sufficient.
Feature Flags Not implemented Useful during Phase 4 to ship incomplete UI components behind flags. A simple FEATURE_FLAGS const object in shared/constants.ts is enough — no LaunchDarkly needed.
Documentation AGENTS.md, interview.md, this report Keep this architecture report updated as the codebase evolves. Consider adding JSDoc to all public API surfaces in the application layer.
